aknDocTypes.json:

All available Akoma Ntoso doc types. see http://schema.akomantoso.com/index.html?type=element&item=akomaNtoso
This never needs to be configured, here only for sanity checks

docTypes.json:

akoma Ntoso types configured currently in the client with SubTypes, this needs to be configured for types you want to make available. 
Every type configured here needs a corresponding workflow configuration, otherwise sanity checks will fail. 

Data: 

data on the client can be only of the types configured in docTypes.json.
